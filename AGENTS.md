# AGENTS.md

Course material for Edward Donner's "Master AI Agentic Engineering" (6 weeks of agent framework labs). It is teaching material, not a single deployable app.

## Layout

Six weekly units; each is a self-contained set of notebooks plus a small `app.py`:

- `1_foundations/` - OpenAI function-calling + Gradio chatbot (`app.py`)
- `2_openai/` - OpenAI Agents SDK labs and `deep_research/`
- `3_crew/` - **five independent CrewAI projects**, each with its own `pyproject.toml` and `uv.lock`: `coder/`, `debate/`, `engineering_team/`, `financial_researcher/`, `stock_picker/`. The root `pyproject.toml` does not cover them.
- `4_langgraph/` - LangGraph Sidekick (`app.py` + `sidekick.py`), uses Playwright
- `5_autogen/` - AutoGen labs + gRPC distributed runtime (`world.py` binds `localhost:50051`)
- `6_mcp/` - MCP trading-floor capstone: `app.py` (Gradio UI) + `trading_floor.py` (long-running scheduler)
- `setup/SETUP-{mac,PC,linux,WSL,node}.md` - OS-specific install guides
- `guides/` - topic-by-topic Jupyter guides
- `*/community_contributions/` - student submissions, ignore unless asked

## Tooling

- Package manager is **`uv`**, not pip. Do not hand-edit the root `pyproject.toml` or `uv.lock`; to upgrade run `uv lock --upgrade`.
- Python 3.12 (pinned in `.python-version`).
- **`crewai` CLI is a uv tool** (separate from the project env): `uv tool install crewai --python 3.12`. Required for anything under `3_crew/`.
- `node` + `npx` are required for `6_mcp` (Brave search and libsql MCP servers). See `setup/SETUP-node.md`.
- `setup/diagnostics.py` and `setup/troubleshooting.ipynb` are the canonical env-check tools.

## Setup from a clean clone

```sh
uv sync                                    # root .venv
uv tool install crewai --python 3.12       # for week 3
```

`.env` lives at the repo root. Apps call `load_dotenv(override=True)`, so CWD does not matter. Minimum: `OPENAI_API_KEY`. Other keys used by specific labs: `GOOGLE_API_KEY` / `GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, `DEEPSEEK_API_KEY`, `OPENROUTER_API_KEY`, `POLYGON_API_KEY`, `BRAVE_API_KEY`, `PUSHOVER_TOKEN` / `PUSHOVER_USER`, `SENDGRID_API_KEY`. Gemini needs both `GOOGLE_API_KEY` and `GEMINI_API_KEY` set to the same value.

## Running things

- Notebooks (`*_lab*.ipynb`): open in the week folder, select the `.venv (Python 3.12)` kernel, run cells. This is the primary learning interface.
- Gradio apps (weeks 1, 4, 6): `uv run app.py` from the week directory.
- CrewAI projects: `cd 3_crew/<project> && crewai install && crewai run`. **Do not** run them via the root `uv run` - they have their own venv.
- `5_autogen` distributed lab: `uv run world.py` (gRPC host on `localhost:50051`).
- `6_mcp` capstone: run `uv run trading_floor.py` in one terminal and `uv run app.py` in another. It spawns several MCP servers via `uv`/`uvx`/`npx` (see `6_mcp/mcp_params.py`).

## Things that don't exist in this repo

- No test suite, no `pytest`, no linter, no formatter, no typechecker, no pre-commit hooks, no CI workflows. Do not propose or run `ruff`/`mypy`/`pytest` - skip the usual verify chain.
- No root `opencode.json`, `.cursorrules`, `.github/`, or existing root `AGENTS.md`.
- The `AGENTS.md` files inside `3_crew/community_contributions/*/` are auto-generated by `crewai create` and only apply to those nested student projects - do not treat them as repo-wide guidance.
